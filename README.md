# FishEye — Accessible Photographer Platform

A modular, accessible web application for an artist platform, built with Vanilla JavaScript and design patterns, as part of a front-end development training program.

## Project Overview

This project was completed as part of a structured front-end development curriculum. The goal was to build a functional prototype of a photographer portfolio platform, starting from a provided codebase, Figma mockups, and JSON data.

The main focus was on **web accessibility** and **code architecture** — writing modular, maintainable JavaScript using design patterns, while ensuring the application is fully usable by everyone, including users relying on assistive technologies.

## Objectives

- Build an accessible web application following best practices (WCAG)
- Develop a modular JavaScript architecture using design patterns
- Write clean, maintainable code
- Handle dynamic data and user interactions with JavaScript

## Key Features

- **Photographer pages** — dynamic rendering of photographer profiles and media from JSON data
- **Lightbox** — accessible media viewer for photos and videos, navigable via keyboard
- **Contact modal** — accessible contact form with focus management
- **Like system** — interactive like counter per media and photographer
- **Full keyboard navigation & screen reader support** — accessibility as a first-class concern throughout

## What I Learned

- Implementing web accessibility standards (ARIA attributes, focus management, keyboard navigation)
- Structuring a JavaScript application with design patterns (Factory, Observer)
- Fetching and rendering dynamic data from a JSON source
- Writing modular, reusable, and maintainable JavaScript code

## Context

This project is part of the **OpenClassrooms Front-End Developer** curriculum. It represents a significant step up in complexity, shifting the focus from visual integration to application architecture and inclusive design.
